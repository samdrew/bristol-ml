## Single highest-leverage cut

If you cut one item to halve this plan's scope, cut **D9's wall-time acceptance test** (the 60k-hour performance bound assertion), because it is the only item that introduces a machine-speed-dependent CI dimension, and its removal does not weaken any of the five intent ACs — bi-temporal correctness (AC-1) is fully covered by the correctness tests in `test_remit.py`, leaving the vectorisation itself in place as an implementation choice without a brittle timing gate around it.
