- Stage 10: `bristol_ml.models.nn.NnMlpModel` — first PyTorch model family behind the Stage 4 `Model` protocol. Small MLP (1 hidden layer × 128 units, ReLU, Adam `lr=1e-3`, `batch_size=32`, `max_epochs=100`, `patience=10` — all Hydra-overridable via `NnMlpConfig`). Hand-rolled training loop at `NnMlpModel._run_training_loop` with patience-based early stopping, best-epoch weight restore, a contiguous 10 %-tail internal validation slice (time-series-safe, not a random split — plan D9), cold-start per fold (plan D8 / Ash & Adams NeurIPS 2020), and a `loss_history_: list[dict[str, float]]` attribute populated per epoch with `{"epoch", "train_loss", "val_loss"}`. Z-score normalisation of features and target on per-fold train statistics, with the four scaler tensors (`feature_mean`, `feature_std`, `target_mean`, `target_std`) registered via `register_buffer(...)` so they ride inside `state_dict()` and round-trip cleanly through `torch.save` / `torch.load` (plan D4). - Stage 8: `bristol_ml.models.scipy_parametric.ScipyParametricModel` — `scipy.optimize.curve_fit` parametric load model behind the Stage 4 `Model` protocol. The functional form is `α + β_heat · HDD + β_cool · CDD + Σ_k (A_k sin + B_k cos)(diurnal) + Σ_j (C_j sin + D_j cos)(weekly)` with Elexon-convention fixed hinges `T_heat = 15.5 °C`, `T_cool = 22.0 °C` (plan D1); `HDD = max(0, T_heat − T)`, `CDD = max(0, T − T_cool)`. Default harmonic counts are `diurnal_harmonics=3` (period 24 h) + `weekly_harmonics=2` (period 168 h), giving 13 free parameters (plan D2). `curve_fit(method="lm", loss="linear", maxfev=5000)` by default (plan D3 + D6) so the covariance-derived 95 % CIs stay rigorously Gaussian; `loss ∈ {"soft_l1", "huber", "cauchy"}` available as CLI override. `p0` derived deterministically from training data inside `fit()` (plan D4 — `α₀ = target.mean()`, `β_heat₀` from sub-10 °C vs above-20 °C mean difference, `β_cool₀` from above-22 °C vs at-17 °C, Fourier coefficients zero) so rolling-origin folds with seasonal training windows do not land far from any reasonable value. `_parametric_fn`, `_derive_p0`, `_build_param_names` are module-level pure functions (plan T2 surprise S2 guard — required for joblib pickle round-trip). `fit` + `predict` both call a `_require_utc_datetimeindex(features, method=...)` static guard mirroring `SarimaxModel` (plan D8). NFR-4 guard emits a loguru WARNING when `pcov` contains `inf` and stores `float("inf")` in the metadata rather than silently producing vacuous CIs (AC-6). Module CLI `python -m bristol_ml.models.scipy_parametric --help` prints config defaults + curve_fit pointer + Stage 6 palette notice (AC-10).
